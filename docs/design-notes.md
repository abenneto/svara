# 设计笔记

这里记录一些“为什么这么做”的取舍，方便日后回顾，也解释一些看起来
不那么显然的实现选择。

## 为什么只用 NumPy / SciPy

项目定位是歌声/语音合成研究的**语音一侧前端**与教学参考。相比拉进
PyTorch/TensorFlow，坚持纯 NumPy/SciPy 带来三个好处：

- **可复现**：没有 GPU、cuDNN 版本、非确定性算子的干扰，固定随机种子即可复现。
- **可读**：每个算法都是几十行能读完的数学，适合对照论文学习。
- **易装**：CI 与本地环境几乎零摩擦，安装秒级完成。

代价是拿不到端到端可学习的表示——这部分交给下游模型，本库只负责稳定、
透明的手工特征与轻量无监督表示。

## F0：为什么给三种方法

没有哪种基频估计器在所有信号上都最好，所以干脆都提供，并用统一的
`estimate_f0(method=...)` 分派：

- 自相关法直观但对倍频/半频错误敏感，因此额外加了抛物线插值细化峰值位置。
- YIN 用累积均值归一化差分（CMNDF）+ 绝对阈值，显著减少八度错误，是默认选择。
- 倒谱法依赖谐波结构，对纯音会失效，但在谐波丰富时给出与前两者一致的结果，
  可作为交叉验证。

清浊音判决没有单独建模，而是让每种方法在“找不到可信峰值”时直接返回 0，
下游用 `f0 > 0` 即可得到浊音掩码，简单且够用。

## 共振峰：LPC 求根而非峰值拾取

对 LPC 预测多项式直接求根，比在 LPC 谱包络上找局部极大更稳：根的角度给出
频率、模长给出带宽，天然成对且有物理意义。我们丢弃带宽过大（衰减太快、
多半是数值根或声门贡献）和频率过低的根，再按频率升序取前若干个。

窗长与阶数是主要的调参点：阶数按经验取 `2 + sr/1000`，大致对应每 1 kHz 一对极点。

## 离散声学单元

`AcousticUnitizer` 借鉴了自监督语音表示里的“离散单元”思路：先在特征上学一个
K-means 码本，再把每帧量化到最近码字。虽然用的是 MFCC 而非自监督特征，但它
提供了两个有用的产物——离散单元序列（可作为下游序列模型输入）和单元词袋
（定长句级表示）。K-means 固定随机种子保证结果确定。

## 已知的粗糙之处
