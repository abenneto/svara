# 架构说明

svara 的整体思路是**一条自底向上的短时分析流水线**：原始波形先被切成帧，
再逐层派生出频谱、倒谱、基频、共振峰，最后汇聚成各类声学表示。每一层只依赖
下一层，模块之间没有环状依赖，方便单独测试与替换。

```
                    ┌─────────────┐
   WAV ── io ──▶    │   framing   │  预加重 / 分帧 / 加窗
                    └──────┬──────┘
                           ▼
                    ┌─────────────┐
                    │  spectral   │  STFT / 功率谱 / 谱图
                    └──────┬──────┘
              ┌────────────┼───────────────┐
              ▼            ▼               ▼
        ┌───────────┐ ┌─────────┐   ┌──────────────┐
        │filterbanks│ │  pitch  │   │   formant    │
        └─────┬─────┘ │ (F0)    │   │ (LPC + 求根) │
              ▼       └─────────┘   └──────────────┘
        ┌───────────┐
        │ features  │  MFCC / LFCC / 谱形状 / 时域 / 差分
        └─────┬─────┘
              ▼
        ┌────────────────────────────┐
        │      representations       │  CMVN / PCA / K-means / 聚合
        └────────────────────────────┘
```

## 分层职责

- **io** —— 唯一接触文件系统的层：WAV 读写、重采样、特征落盘。其余模块只处理内存中的数组。
- **framing** —— 把一维信号变成 `(帧数, 帧长)` 的二维数组。用 `sliding_window_view`
  做零拷贝滑窗，是后续几乎所有分析的公共入口。
- **spectral** —— 短时傅里叶变换及其派生量，向上游提供复数谱 / 功率谱。
- **filterbanks / features** —— 把频谱映射到感知相关或去相关的特征空间。
- **pitch / formant** —— 两条相对独立的分析支线，分别刻画声源（基频）与声道（共振峰）。
- **representations** —— 在帧级特征之上做归一化、降维、离散化与聚合，产出下游可直接使用的表示。

## 约定

- 所有对外函数以 `float64`、单声道、`[-1, 1]` 归一化信号为输入。
- 时长相关参数一律以**样点**为单位；频率以 **Hz** 为单位。
- 帧级特征统一为 `(n_frames, n_features)` 的行优先布局。
- 计算过程不做隐式重采样——采样率由调用方显式传入并保持一致。

## 可测试性

由于不依赖外部音频，所有算法都能用**合成信号**验证：正弦验证 F0，
源–滤波器合成的元音验证共振峰，随机高斯验证归一化 / 聚类的统计性质。
这保证了 CI 完全离线、且结果可复现。
